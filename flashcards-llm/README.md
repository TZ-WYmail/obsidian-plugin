# 闪卡 LLM（自改中文版）

这是一个面向个人 Obsidian + Anki 工作流修改的 AI 制卡插件。插件会把 Obsidian 中选中的文本或 `==高亮内容==` 发送给兼容 OpenAI 接口的大语言模型，生成 Anki 卡片，并通过 AnkiConnect 直接导入指定牌组。

## 当前自改能力

- 选中文本后自动显示浮动按钮：`问答卡`、`知识点卡` 和 `划重点`。
- `划重点` 会使用 Obsidian 原生 `==高亮==` 加隐藏标记记录重点 ID，不再用 HTML 包裹破坏列表结构。
- `划重点` 默认快捷键为 `Ctrl+Alt+H`，也可以在 Obsidian 快捷键设置中修改。
- 提供 `修复旧版重点标记为安全格式` 命令，可把旧版 `<u ...>` 标记迁移为新格式。
- 支持生成问答卡片，输出 `START / Basic / Q: / A: / END` 结构。
- 支持生成知识点卡片，优先输出 `START / Cloze / {{c1::...}} / END` 结构。
- 支持批量命令把每一段已划重点内容生成对应卡片；重复执行时会按重点原文 hash 自动去重。
- 批量知识点卡片使用双面 Basic 卡：正面为 AI 生成的回忆提示，背面为笔记原话。
- 点击生成后会立即打开预览弹窗，模型输出会流式写入预览框，不再等待完整回复后才显示。
- 保存前提供预览与编辑弹窗，生成完成后可修改模型输出再确认生成。
- LLM 请求支持最多 3 次连接重试：网络异常、`429`、`408` 和服务端 `5xx` 会自动退避重试；`400/401/403` 等配置错误会直接报错，避免无效等待。
- 内置问答质量评估：如果模型生成 `这段内容的核心知识点是什么？`、`上述内容说明了什么？` 等泛化题干，会自动判定为不合格并改写成包含原文术语或标题锚点的具体问题。
- 内置兜底问答重写：当模型返回空内容、非问答格式或不可复习的题干时，会根据标题和列表项生成可回答的问题，并压缩答案，避免整段笔记被塞进同一张卡。
- 点击确认后会在当前笔记同目录创建 `笔记名-card` 文件夹，并把本次卡片保存为独立 Markdown 文件。
- 保存成功后会通过 AnkiConnect 直接导入 Anki，不再依赖 Export to Anki 扫描命令。
- 设置页新增 `Anki 直连同步` 区域，可测试连接、刷新牌组、选择牌组命名方式、配置 Basic/Cloze 模板字段。
- 设置页新增批量按钮：`批量问答`、`批量知识点`、`导入当前卡片文件夹`、`开始复习`。
- 支持在 Obsidian 内直接复习 Anki：读取当前笔记同名牌组的待复习/新卡，翻面后用 `重来`、`困难`、`良好`、`简单` 写回 Anki 调度记录。
- 左侧 Ribbon 侧边栏新增复习按钮，点击即可打开当前笔记牌组的复习界面。
- 卡片生成预览弹窗和 Obsidian 内复习弹窗支持拖拽右下角调整大小。
- 流式解析已增强：兼容 `data:` 无空格、多个 SSE 数据行和裸 JSON；流式响应为空会自动重试非流式请求，仍为空时会生成兜底卡片。
- 支持兼容 OpenAI 的接口地址，可配置 DeepSeek、通义千问、智谱、月之暗面、本地 Ollama 等供应商。
- 插件界面、命令、通知、设置页和内置提示词已改为中文，便于确认当前加载的是自改版本。

## 推荐设置

在 Obsidian 插件设置中填写：

- `API 基础地址`：供应商接口根地址，例如 `https://api.deepseek.com`。
- `API 请求路径`：通常填写 `/v1/chat/completions`。
- `鉴权请求头`：通常填写 `Authorization`。
- `鉴权前缀`：通常填写 `Bearer `，注意末尾空格。
- `API Key`：填写供应商控制台生成的密钥。
- `模型名称`：例如 `deepseek-chat`、`qwen-plus`、`glm-4-plus`、`moonshot-v1-8k`。
- `输出格式`：保持 `Obsidian_to_Anki（推荐）`。
- `AnkiConnect 地址`：默认 `http://127.0.0.1:8765`。
- `牌组命名方式`：默认 `跟随当前笔记名`，例如当前打开 `算法.md` 时会导入到 Anki 牌组 `算法`。
- `固定目标牌组`：只有切换为 `使用固定牌组` 时才生效，可点 `刷新牌组` 从 Anki 自动读取。
- `自动导入到 Anki`：建议开启。确认生成或批量生成后会自动写入 Anki。
- `每轮复习上限`：Obsidian 内复习一次最多读取的卡片数，默认 20。
- `问答模板/字段`：默认 `Basic`、`Front`、`Back`。
- `填空模板/字段`：默认 `填空题`、`文字`、`背面额外`。

## 使用方式

1. 在 Obsidian 源码模式中选中需要记忆的内容。
2. 等待选区上方出现浮动按钮。
3. 点击 `问答卡`，生成 Q/A 形式卡片。
4. 点击 `知识点卡`，生成知识点或填空形式卡片。
5. 点击 `划重点`，把选中文本标记为后续可批处理的重点。
6. 预览弹窗会立即出现，并实时显示模型正在生成的内容。
7. 生成完成后，在预览弹窗中检查和修改结果。
8. 点击 `确认生成并同步`。
9. 插件会先保存到当前笔记同目录的 `笔记名-card` 文件夹，再通过 AnkiConnect 导入与当前笔记同名的 Anki 牌组。

预览弹窗不会再提供“插入当前笔记”按钮，避免卡片保存位置不明确。

如果没有选中文本，命令入口会优先提取当前笔记中的 `==高亮内容==` 作为制卡材料；如果也没有高亮，则使用整篇笔记作为上下文。

## 批量重点制卡

先选中文本点击浮窗里的 `划重点`，源码中会变成类似：

```markdown
- ==需要记忆的原文== %% flashcards-llm-key-point:id=... %%
```

如果选中多行内容，会使用隐藏的块标记包住多行，并对每行内容做 `==高亮==`。列表、标题和引用的 Markdown 结构会保留在高亮外侧，所以 `-` 仍然能正常渲染为列表点。

然后在 Obsidian 命令面板运行：

- `批量生成重点问答卡片`：每段重点生成一张问答卡。
- `批量生成重点知识点卡片（背面原话）`：每段重点生成一张双面知识点卡，背面保留笔记原话。
- `导入当前笔记卡片文件夹到 Anki`：把当前笔记同目录 `笔记名-card` 文件夹中的卡片批量导入 Anki。
- `测试 AnkiConnect 连接`：检查当前 AnkiConnect 地址、牌组和模板是否可读取。
- `在 Obsidian 中复习当前笔记牌组`：直接打开复习弹窗，不需要切换到 Anki 界面。
- `修复旧版重点标记为安全格式`：把历史 `<u class="flashcards-llm-key-point" ...>` 标记迁移成新格式。

批量生成会保存到当前笔记同目录的 `笔记名-card` 文件夹。每段重点会记录 `source_hash`，再次执行批量命令时，已经生成过的重点会自动跳过；导入 Anki 时也会使用 AnkiConnect 的重复检测避免重复添加。

## Obsidian 内复习

打开一篇笔记后，点击 Obsidian 左侧 Ribbon 的复习按钮，或运行命令 `在 Obsidian 中复习当前笔记牌组`，也可以在插件设置页点击 `开始复习`。插件会按当前牌组命名方式定位 Anki 牌组，默认是当前笔记同名牌组。

复习弹窗会从 AnkiConnect 读取 `is:due` 和 `is:new` 卡片，点击 `显示答案` 后选择 `重来`、`困难`、`良好` 或 `简单`。评分会通过 AnkiConnect 的 `answerCards` 写回 Anki，因此仍使用 Anki 当前启用的调度算法和 FSRS 参数。

## 输出格式

问答卡片示例：

```markdown
<!-- CARD -->
START
Basic
Q: FSRS 的作用是什么？
A: FSRS 是 Anki 的间隔重复调度算法，用于根据记忆状态安排复习时间。
END
```

填空卡片示例：

```markdown
<!-- CARD -->
START
Cloze
FSRS 是 Anki 的 {{c1::间隔重复调度算法}}。
END
```

这些格式会被插件内置解析器读取，并通过 AnkiConnect 写入当前笔记同名牌组；如果设置为固定牌组，则写入设置页指定的牌组。Export to Anki 不再是插件生成卡片的必需依赖，只保留为旧工作流兼容参考。
